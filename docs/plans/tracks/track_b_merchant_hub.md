# Track B: Merchant Hub & Registry UI
**Branch Name:** `feat/merchant-hub-ui`
**Base Branch:** `base-integration` (Consolidated)

## 1. Objective
To transform the Hub into a "Commercial Terminal" that visualizes the full range of the Agent's financial power. It must support listing products, accepting various payment strategies (including Starpay Cards), and displaying the complex status of cross-chain operations.

## 2. Core Components

### 2.1. The Merchant Registry (On-Chain)
Enhanced to support sophisticated service listings.
*   **Path:** `onchain/programs/xb77_registry/`
*   **New Data:** Add `supported_methods` bitmask to Merchant accounts (e.g., accepts `PRIVACY_CASH | STARPAY_CARD | SILENTSWAP_BRIDGED`).
*   **Products:** Define "Virtual Goods" (e.g., "AWS Credits", "Confidential Compute") that justify using Starpay or ShadowWire.

### 2.2. Hub UI: The Agent Dashboard (Client)
*   **Strategy Selector:** A UI component that lets the user configure the Agent's "CFO Settings" (e.g., "Always use Starpay for amounts < $50").
*   **Card Viewer:** A secure component to display Starpay Virtual Card details (PAN, CVV) generated by Track A.
*   **Cross-Chain Monitor:** A status panel showing SilentSwap bridge operations ("Moving to Base... 40%").

### 2.3. Hub UI: The Merchant Dashboard (Vendor)
*   **Sales Feed:** Real-time list of incoming payments.
*   **Multi-Rail Recognition:** Visual distinction between a "Privacy Cash" payment and a "Starpay Card" settlement.
*   **Invoice Generator:** Button to manually issue a **Compressed Receipt** (Light Protocol) if one wasn't auto-generated.

## 3. Integration Points

### Input from Track A (Agent Core)
*   The Hub uses the `AutonomousCFO` class from the SDK to execute actions.
*   It subscribes to SDK events to update the UI (e.g., "Card Created", "Swap Initiated").

### Input from Track C (Infra Listener)
*   **Data Feeds:** The Hub does *not* query the chain directly for complex history. It queries the **MCP API** (Track C) to get the aggregated history of all rails (ShadowWire + Starpay + SilentSwap).
*   **Real-Time:** WebSockets from Track C update the Sales Feed.

## 4. Execution Plan
1.  **Registry Upgrade:** Update Rust program to support payment methods metadata.
2.  **Product UI:** Build the "Marketplace" view.
3.  **Checkout Flow:** Connect the "Buy" button to `agent.pay(strategy)`.
4.  **Starpay UI:** Build the Virtual Card display component.
5.  **History Integration:** Connect the Activity Feed to the Track C API.
