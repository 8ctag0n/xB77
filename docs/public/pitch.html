01 · The problem

Cloud-bound agents are dead agents.

AI agents today are cloud-bound clients. They depend on third-party LLM APIs, leak strategic data to vendors, and die the moment a key gets rotated or a provider changes terms.

There is no financial OS designed for entities that need to reason in private, transact at machine speed, and prove compliance without exposing alpha.

A swarm of a million agents on Web2 rails is a swarm with a million single points of failure.

A merchant agent boots locally, claims a .sol identity via SNS, parses natural-language directives through a local QVAC brain, validates them against a cryptographically-enforced Constitution, settles intents on a MagicBlock Private Ephemeral Rollup, and emits a Noir "Ghost Receipt" that proves tax compliance to Solana mainnet without revealing amounts or counterparties.

The full path — directive → reasoning → settlement → ZK receipt — works air-gapped except for the final settle.

Impossible without these primitives.

  • MagicBlock PER — sub-millisecond execution rail with mainnet-anchored settlement; agentic commerce needs this throughput, not L2 rollup economics.
  • SNS (.sol) — agents need a name they can own and a counterparty whitelist they can verify offline; SNS resolves cached.
  • Solana mainnet — the ZK-Judge contract verifies Noir Plonk proofs as the final source of truth.
  • Anchor — onchain state anchor for the Constitution and proof commitments.
04 · Technical highlights

Zig core. Local brain. ZK receipts.

  • Zig core (build.zig, core/) — memory-safe, no-runtime engine: P2P gossip (Agent Wire Protocol), gRPC, local intelligence layer.
  • QVAC brain (core/intelligence/brain.zig) — Llama.cpp-bound RAG that parses directives and enforces a Constitution before any packet leaves the host.
  • MagicBlock rail (core/chain/magicblock.zig) — PER session adapter for sub-second intent settlement.
  • Noir circuits (circuits/) — Plonk proofs for tax-compliance receipts; verifier deployed onchain as a ZK-Judge.
  • WASM Gateway (gateway/) — brutalist dashboard served from the agent itself for in-browser selective-disclosure auditing.
  • SNS gating (core/security/identity.zig) — counterparty whitelisting at the Constitution layer, not the RPC layer.
  • Tether WDK (core/security/wdk.zig) — non-custodial vault for agent keys.
